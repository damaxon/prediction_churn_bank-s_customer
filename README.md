# Customer Churn Prediction

Проект машинного обучения для предсказания оттока клиентов банка. Модель обучена с использованием Пайплайна scikit-learn с подбором threshold чтобы оптимизировать F1-метрику.

## 1. Бизнес-проблема
- Отток клиентов является одной из ключевых проблем для современных банков, так как любой банк хочет понимать удержание своих клиентов. Привлечение новых клиентов обходится значительно дороже, чем удержание существующего.

- Цель проекта - построить модель, которая предсказывает вероятность ухода клиента, чтобы бизнес мог заранее обнаружить клиентов с высоким риском и применить какие-либо меры для удержания.

- Посколько пропуск клиента, который действительно уйдет, более критичен, чем случайно применить меры на клиента, который не собирался уходить, приоритетными метриками в выборе лучшей модели являются **Recall** и **F1-score**, а не Accuracy или ROC_AUC.

## 2. Датасет
    Датасет содержит информацию о клиентах банка:

**Числовые признаки**:
- CreditScore
- Age
- Tenure
- Balance
- NumOfProducts
- HasCrCard
- IsActiveMember
- EstimatedSalary

**Категориальные признаки**:
- Geography
- Gender
- Card Type

**Целевая переменная**:
- `Exited` (1 - клиент ушел, 0 - остался)

**Распределение классов**:
- ~20% клиентов ушли
- ~80% клиентов остались

Наблюдается дисбаланс классов.

## 3. Подход к моделированию

### 1. Предобработка данных

Используется `ColumnTransformer`:
- Для числовых признаков:
    - Median Imputation
    - StandartScaler
- Для категориальных признаков:
    - Constant Imputation
    - OneHotEncoder

### 2. Pipeline

    Модель и препроцессинг объединены в единый Pipeline, что предотвращает утечку данных и упрощает воспроизводимость.

### 3. Подбор гиперпараметров

Используется:
- GridSearchCV
- StratifiedKFold
- Метрика отбора модели - `ROC_AUC`

ROC_AUC выбран для устойчивой оценки качества при дисбалансе классов.

### 4. Оптимизация порога классификации

После выбора лучшей модели применяется:

    TunedThresholdClassifierCV

Цель - оптимизация **F1-score** за счет подбора оптимального порога вероятности.

Стандартный порог: 0.5

Оптимальный порог: 0.337

### Результаты
Лучшая модель: `Gradient Boosting`

| Метрика   | Стандартный порог | Оптимизированный порог |   |   |
|-----------|-------------------|------------------------|---|---|
| Accuracy  | 0.872             | 0.854                  |   |   |
| Precision | 0.80              | 0.65                   |   |   |
| Recall    | 0.49              | 0.61                   |   |   |
| F1        | 0.61              | 0.663                  |   |   |
| ROC_AUC   | 0.877             | 0.877                  |   |   |


**Вывод**:
Подбор порога позволил увеличить Recall и F1-score, что делает модель более подходящей для задачи прогнозирования оттока клиентов.

ROC_AUC при этом остался неизменным, так как изменение threshold не влияет на ранжирование вероятностей.

## 7. Пример предсказания

```python
from src.predict import load_model, predict

model = load_model("model/model.joblib")

sample = {
    "CreditScore": 600,
    "Age": 40,
    "Tenure": 5,
    "Balance": 60000,
    "NumOfProducts": 2,
    "HasCrCard": 1,
    "IsActiveMember": 1,
    "EstimatedSalary": 50000,
    "Geography": "France",
    "Gender": "Male",
    "Card Type": "Silver",
}

prediction, probability = predict(model, sample)

print("Prediction:", prediction)
print("Probability:", probability)
```
